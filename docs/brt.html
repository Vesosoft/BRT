BRT - Browser Reader Mode Compatibility Test

1. Introduction

BRT (Browser Reader Mode Compatibility Test) is an open unofficial methodology for quickly checking whether web content can be comfortably read using built-in accessibility features of modern browsers.

The methodology was created to help users, testers, and developers identify barriers that may make access to the main information on a web page more difficult.

BRT does not evaluate the design or quality of a website's content and does not aim to criticize its creators. The focus is on whether the main content remains accessible and convenient to read.

2. Principles

BRT checks the accessibility of the main content, not the visual appearance of the website.

BRT does not evaluate website design, visual layout, or the presence of advertisements.

The goal is to check whether users can quickly and conveniently reach the main information through browser accessibility features.

Advertisements, pop-ups, side panels, and other additional elements can make reading more difficult. BRT checks how Reader Mode presents the main content and whether it remains convenient to use.

BRT does not evaluate the content of an article itself, but only the accessibility of its presentation through browser reading features.

3. Purpose of the Methodology

The purpose of BRT is to provide an easy and understandable way to check the accessibility of web content through built-in accessibility features of modern browsers.

The methodology allows users without technical knowledge to determine whether the main information on a web page can be read comfortably using Reader Mode and other assistive settings.

BRT does not require specialized software or programming skills. Only a browser with available readability improvement features is required.

The main goal is to identify barriers that may limit users' access to information on the internet.

4. What is Reader Mode?

Reader Mode (Reader View) is a browser feature that presents the main content of a web page in a more readable format by removing or reducing distracting elements such as advertisements, menus, and additional content blocks.

This feature can be used for easier reading of articles and long texts, as well as for checking web content accessibility using BRT.

4.1. Google Chrome

In Google Chrome, Reader Mode can be accessed from the three-dot menu (⋮) at the top of the browser.

Select:

  • Menu (⋮)
  • Show reading mode

4.2. Mozilla Firefox

In Mozilla Firefox, Reader View can be activated using the rectangular icon on the right side of the address bar (URL field) when the browser recognizes the page as suitable for reading.

If the icon is not displayed, this does not automatically mean that the page is inaccessible. Firefox may simply not recognize the content as suitable for Reader View.

5. Required Tools

BRT does not require special software or technical knowledge. Only a device with internet access and a modern web browser is required.

To perform a test, the following are required:

  • A computer, tablet, or smartphone.
  • A web browser with Reader Mode functionality, if available.
  • A web page to be tested.

When recording test results, it is recommended to include:

  • the browser used;
  • the browser version;
  • the test date.

This information helps make test results verifiable and comparable over time.

6. Testing Procedure

BRT is performed through a practical check of a web page using browser accessibility features.

Step 1: Opening the page

Open the web page you want to test in the selected browser.

Step 2: Activating Reader Mode

Enable Reader Mode (Reader View) if the browser provides this feature.

Step 3: Checking the content

Check whether the main information on the page is available and clearly presented. Pay attention to:

  • title and main text;
  • content completeness;
  • text readability;
  • presence of distracting elements.

Step 4: Determining the result

After the check, determine the result according to the PASS, PARTIAL, or FAIL criteria.

7. Evaluation Criteria

BRT uses three result levels for checking the accessibility of the main content. The result is determined by how well the information can be read through browser accessibility features.

🟢 PASS

The main content is accessible and can be read comfortably.

  • The text is extracted correctly.
  • The main information is available.
  • The content structure remains understandable.
  • There are no significant barriers to reading.

🟠 PARTIAL

The content is partially accessible, but there are problems that may make it more difficult for the user.

  • Some parts of the content are missing.
  • Some elements are presented incorrectly.
  • Reading is possible, but not convenient.

🔴 FAIL

The main content cannot be effectively used through Reader Mode or other tested accessibility features.

  • The main text is not extracted.
  • Unrelated content is displayed.
  • The information remains inaccessible to the user.

The result is determined by the accessibility of the main content, not by the visual appearance of the website.
