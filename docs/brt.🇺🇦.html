BRT - Browser Reader Mode Compatibility Test

1. Вступ

BRT (Browser Reader Mode Compatibility Test) — це відкрита неофіційна методика для швидкої перевірки того, чи можна зручно читати вебконтент за допомогою вбудованих функцій доступності сучасних браузерів.

Методика створена для допомоги користувачам, тестувальникам і розробникам у виявленні перешкод, які можуть ускладнювати доступ до основної інформації на вебсторінці.

BRT не оцінює дизайн або якість вмісту вебсайту та не має на меті критикувати його авторів. Основна увага приділяється тому, чи залишається основний вміст доступним і зручним для читання.

2. Принципи

BRT перевіряє доступність основного вмісту, а не зовнішній вигляд вебсайту.

BRT не оцінює дизайн вебсайту, візуальне оформлення або наявність реклами.

Мета полягає в тому, щоб перевірити, чи може користувач швидко та зручно отримати доступ до основної інформації за допомогою функцій доступності браузера.

Реклама, спливаючі вікна, бічні панелі та інші додаткові елементи можуть ускладнювати читання. BRT перевіряє, як режим читання представляє основний вміст і чи залишається він зручним для використання.

BRT не оцінює зміст самої статті, а лише доступність її представлення за допомогою функцій читання браузера.

3. Мета методики

Мета BRT — надати простий і зрозумілий спосіб перевірки доступності вебконтенту за допомогою вбудованих функцій доступності сучасних браузерів.

Методика дозволяє користувачам без технічних знань визначити, чи можна зручно прочитати основну інформацію на вебсторінці за допомогою режиму читання та інших допоміжних налаштуван

BRT не потребує спеціалізованого програмного забезпечення або навичок програмування. Потрібен лише браузер із функціями покращення читабельності.

Основна мета — виявлення перешкод, які можуть обмежувати доступ користувачів до інформації в інтернеті.

4. Що таке Reader Mode?

Reader Mode (Reader View) — це функція браузера, яка представляє основний вміст вебсторінки у зручнішому для читання форматі, видаляючи або зменшуючи відволікаючі елементи, такі як реклама, меню та додаткові блоки вмісту.

Цю функцію можна використовувати для зручнішого читання статей і довгих текстів, а також для перевірки доступності вебконтенту за допомогою BRT.

4.1. Google Chrome

У Google Chrome режим читання можна знайти через меню з трьома крапками (⋮) у верхній частині браузера.

Виберіть:

  • Меню (⋮)
  • Показати режим читання

4.2. Mozilla Firefox

У Mozilla Firefox Reader View можна активувати за допомогою прямокутної іконки праворуч від адресного рядка (URL-поля), коли браузер розпізнає сторінку як придатну для читання.

Якщо іконка не відображається, це не означає автоматично, що сторінка є недоступною. Firefox може просто не розпізнати вміст як придатний для Reader View.

5. Необхідні інструменти

BRT не потребує спеціального програмного забезпечення або технічних знань. Потрібен лише пристрій із доступом до інтернету та сучасний веббраузер.

Для проведення тесту необхідні:

  • Комп’ютер, планшет або смартфон.
  • Веббраузер із функцією Reader Mode, якщо вона доступна.
  • Вебсторінка для перевірки.

Під час запису результатів тестування рекомендується зазначати:

  • використаний браузер;
  • версію браузера;
  • дату проведення тесту.

Ця інформація допомагає зробити результати тестування перевірюваними та порівнюваними в майбутньому.

6. Процедура тестування

BRT виконується шляхом практичної перевірки вебсторінки за допомогою функцій доступності браузера.

Крок 1: Відкриття сторінки

Відкрийте вебсторінку, яку потрібно перевірити, у вибраному браузері.

Крок 2: Активація режиму читання

Увімкніть режим читання (Reader Mode / Reader View), якщо браузер підтримує цю функцію.

Крок 3: Перевірка вмісту

Перевірте, чи основна інформація на сторінці доступна та представлена зрозуміло. Зверніть увагу на:

  • заголовок і основний текст;
  • повноту вмісту;
  • читабельність тексту;
  • наявність елементів, що заважають читанню.

Крок 4: Визначення результату

Після перевірки визначається результат відповідно до критеріїв PASS, PARTIAL або FAIL.

7. Критерії оцінювання

BRT використовує три рівні результатів для перевірки доступності основного вмісту. Результат визначається залежно від того, наскільки добре інформація може бути прочитана за допомогою функцій доступності браузера.

🟢 PASS

Основний вміст доступний і може бути зручно прочитаний.

  • Текст правильно витягується.
  • Основна інформація доступна.
  • Структура вмісту залишається зрозумілою.
  • Немає суттєвих перешкод для читання.

🟠 PARTIAL

Вміст частково доступний, але є проблеми, які можуть ускладнювати використання для користувача.

  • Відсутні частини вмісту.
  • Деякі елементи представлені неправильно.
  • Читання можливе, але не є зручним.

🔴 FAIL

Основний вміст не може бути ефективно використаний через Reader Mode або інші перевірювані функції доступності.

  • Основний текст не витягується.
  • Відображається незв’язаний вміст.
  • Інформація залишається недоступною для користувача.

Результат визначається доступністю основного вмісту, а не зовнішнім виглядом вебсайту.

8. Приклад тестування

8.1 Вебсайт, що тестується

Як приклад практичного застосування BRT було використано вебсайт Meteo Balkans.

Мета тесту — перевірити, чи може основний вміст бути доступним через Reader Mode у різних браузерах.

8.2 Використані браузери

  • Google Chrome
  • Mozilla Firefox
  • Microsoft Edge
  • Opera

8.3 Результати

Google Chrome:

  • Reader Mode: FAIL 🔴
  • Заголовок статті витягується, але основний вміст недоступний.

Mozilla Firefox:

  • Reader View: FAIL 🔴
  • Reader View активується, але основний вміст статті не відображається.

Microsoft Edge:

  • Тест №001 – Meteo Balkans
  • Спостереження:

AI-асистент браузера надав спрощену версію вмісту статті після того, як користувач запросив режим читання.

Результат:
Це не є результатом тестування Reader Mode / Reader View. AI-асистент надав альтернативний спосіб представлення вмісту.

Посилання на детальне спостереження: Test #001 - Browser Accessibility

8.4 Висновок

Одна й та сама вебсторінка може давати різні результати в різних браузерах. Тому BRT розглядає кожну комбінацію браузера та вебсторінки як окремий тестовий випадок.

8.5 Тест №2 – Miele

8.5.1 Вебсайт, що тестується

Вебсайт: Miele Bulgaria

Мета тесту — перевірити, чи може основний вміст бути доступним через Reader Mode у різних браузерах.

8.5.2 Google Chrome

Звичайний вигляд:

Reader Mode:

  • Результат: PASS 🟢
  • Основний вміст успішно відображається.

8.5.3 Mozilla Firefox

  • Результат: FAIL 🔴
  • Firefox не пропонує Reader View для цієї сторінки.

8.5.4 Висновок

У цьому тесті Google Chrome успішно надає Reader Mode і відображає основний вміст. Mozilla Firefox не пропонує Reader View для тієї самої сторінки.

Цей приклад показує, що одна й та сама вебсторінка може оброблятися по-різному різними браузерами. Тому BRT рекомендує виконувати перевірку за допомогою кількох браузерів.
