BRT (Browser Reader Mode Compatibility Test) — це відкрита неофіційна методика для швидкої перевірки того, наскільки вебсайт зручний для читання за допомогою функцій доступності браузера.

Мета BRT — дати можливість кожній людині без спеціальних технічних знань перевірити, чи є основний вміст вебсайту доступним.

## Чому потрібен BRT?

Багато людей використовують допоміжні функції, такі як:

- режим читання (Reader Mode / Reader View);
- збільшення тексту;
- зміна контрастності;
- темний режим;
- інші налаштування для зручнішого читання.

Якщо вебсайт погано працює з цими функціями, частина користувачів може втратити доступ до його вмісту.

## Що перевіряє BRT?

BRT перевіря:

- чи правильно витягується основний текст;
- чи відображається повна стаття;
- чи функції читання видаляють елементи, що заважають читанню;
- чи можна комфортно читати текст;
- чи є проблеми із зображеннями та додатковим вмістом.

## Тестовані браузери

BRT можна тестувати за допомогою різних браузерів із функціями доступності.

Поточні тестові середовища:

- Google Chrome — протестовано Reader Mode;
- Mozilla Firefox — протестовано Reader View;
- Microsoft Edge — протестовано функції доступності; доступність Reader Mode залежить від сторінки та версії;
- Opera — протестовано функції доступності; доступність Reader Mode залежить від сторінки та версії.

## Реальні тестові випадки

BRT містить задокументовані тести доступності, проведені на реальних вебсайтах.

Приклади:

- Test #001 - Meteo Balkans  
  Reader Mode можна активувати, але основний вміст не витягується.

- Test #002 - Miele  
  Chrome Reader Mode працює. Firefox Reader View недоступний.

## Принцип

BRT не є інструментом для оцінювання або критики вебсайтів.

Це інструмент для виявлення перешкод для читачів і покращення доступності вебконтенту.

Цей проєкт створено як відкриту ініціативу для покращення доступності вебконтенту.
