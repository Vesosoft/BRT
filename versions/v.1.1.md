# BRT 1.1

Current release version

## Overview

BRT (Browser Reader Mode Compatibility Test) 1.1 improves the project structure, documentation organization, and website accessibility.

This version introduces a clearer separation between project documentation and the public website hosted through GitHub Pages.

The methodology continues to focus on evaluating whether the main content of a web page remains accessible and readable through browser accessibility features such as Reader Mode / Reader View.

BRT originally started as Bulgarian Reader Test.
Starting with version 1.1, BRT is expanded as Browser Reader Mode Compatibility Test.

## Changes in BRT 1.1

- Moved website files to the "/docs" directory for GitHub Pages deployment
- Separated GitHub project documentation from website content
- Improved project structure and file organization
- Added clearer documentation structure and paths
- Prepared the project for future test cases and additional accessibility guides
- Improved initial multilingual documentation support

## Project Structure

BRT 1.1 uses the following organization:

- "README.md" — project overview and developer documentation
- "docs/" — public website content
- "docs/test-cases/" — practical accessibility tests
- "docs/assets/images/" — screenshots and visual resources
- "versions/" — version history documentation
